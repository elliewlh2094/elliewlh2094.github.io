# Ellie's Robot Notebook 第一版執行計畫

## 目前狀態

這個專案目前已經具備第一版公開網站的基本形態。網站使用 Astro 建立靜態頁面，內容以 Markdown 維護，部署流程維持 GitHub Pages。

相關文件：

- `README.md`：日常維護流程、內容模板與指令入口。
- `AGENTS.md`：維護規則、內容模型與驗證原則。
- `docs/technical-troubleshooting-guide.md`：建置、驗證與內容異常的排查方式。

### 已完成

- 已建立首頁、關於頁、專案列表頁、專案詳細頁、文章列表頁與文章詳細頁。
- 已建立 `longform-demo` 維護展示頁，供草稿內容與長篇排版檢查使用。
- 已使用內容集合（content collections）管理 `src/content/blog` 與 `src/content/projects`。
- 已完成草稿排除機制，`draft: true` 的內容不會出現在正式頁面。
- 已建立 `npm run verify`，會依目前 `draft: false` 的內容動態檢查建置輸出、草稿排除、封面資源與預覽頁面。
- 已建立 GitHub Pages 部署流程，`.github/workflows/deploy.yml` 會在推送到 `main` 時執行 `npm ci` 與 `npm run build`。
- 已有基本頁面中繼資料（metadata），包含頁面標題、描述、Open Graph（開放圖譜）標題、描述、網址與 `noindex` 支援。
- 首頁、卡片、導覽列、頁尾與內容頁已有一致的第一版樣式。
- 網站目前保留混合定位，題材包含機器人研究、Side project、AI 工具觀察與生活內容。

### 目前公開內容

- 公開專案 1 項：
  - `行為樹視覺化工具`
- 公開文章 2 篇：
  - `代理外殼的剖析（The Anatomy of an Agent Harness）`
  - `薄外殼，厚技能（Thin Harness, Fat Skills）`
- 其他專案與文章目前維持草稿狀態。
- 第一版完成前，內容重點以整理現有公開頁面與發布品質為主，暫時不再新增或大幅擴充文章內容。

### 已知現況與限制

- GitHub 連結已存在於頁尾。
- 聯絡信箱仍是暫用占位內容，尚未定稿。
- 首頁、關於頁、部落格列表頁與頁尾文案仍保留較大範圍的混合敘述，和目前公開內容集中在研究工具與翻譯文章的狀態之間還有落差。
- 公開專案與公開文章的封面仍使用暫用圖片或截圖，內容敘事仍可再整理。
- 本機 `npm run build` 與 `npm run verify` 已通過，但線上部署結果尚未完成人工確認。

### 計畫調整判斷

- 計畫方向本身仍成立，網站定位與技術選擇沒有偏離到需要重做。
- 計畫粒度需要收斂。原本文件把許多遠期構想寫成近期待辦，會稀釋第一版真正要完成的事項。
- 第一版接下來的重點是發布品質、文案一致性、公開內容完成度與部署確認，不是功能擴張。

## 第一版待完成

- 補齊 2 篇公開翻譯文章的封面與整理品質，至少讓公開頁面在視覺與說明上達到一致。

### 第一版不列為阻塞項

- 站點地圖（sitemap）、RSS、`404` 頁與更完整的搜尋引擎最佳化（SEO）。
- 履歷頁、經歷頁、技能頁、聯絡頁下載入口。
- 搜尋、分類入口、系列文章機制。

## 後續可能方向

這些項目保留為第二階段之後的方向，不列入第一版近期待辦：

- 站點地圖、RSS、`404` 頁、社群預覽圖與更完整的搜尋引擎最佳化。
- 更完整的內容維護文件，包含欄位說明、圖片規則與編修流程。
- 搜尋、分類、系列文章等內容導覽功能。
- 履歷、經歷、技能或獨立聯絡頁等個人品牌延伸頁面。
- 視第一版穩定度再評估是否公開其他草稿專案與文章。
- 視 Markdown 維護流程是否穩定，再決定是否需要導入內容管理系統。

## 假設與限制

- 第一版的核心目標是穩定公開發布，而不是再擴充功能面。
- 第一版公開內容暫時集中在研究工具與翻譯文章，網站整體定位仍維持混合題材方向。
- 草稿內容與 `longform-demo` 仍保留作為維護與驗證用途。
- 目前不調整公開路由、內容模型、部署架構與驗證腳本的基本設計。
- 第一版完成前，內容新增與大規模擴張不是當前重點。
