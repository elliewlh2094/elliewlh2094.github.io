這是第一版部落格範例文章。內容目前是佔位資料，之後可以替換成實際工作筆記。

## 核心想法

個人網站需要長期維護。把文章與專案都放在 Markdown 檔案中，可以降低更新成本，也能讓內容保留在 Git 版本紀錄裡。

## 更新流程

1. 在 `src/content/blog` 新增 Markdown 檔案。
2. 填寫標題、摘要、日期、標籤與草稿狀態。
3. 在內文寫下完整文章。
4. 推送到 GitHub 後由 GitHub Actions 自動發布。

這個流程適合逐步累積專業文章與專案復盤。
